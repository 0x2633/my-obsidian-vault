## SCRAMPER 策略優化框架

是一套由艾伯列（Bob Eberle）提出的==創意啟發檢查表==。

將 SCRAMPER 拆解並應用於交易策略開發：

| **字母** | **行動 (Action)**             | **金融/交易策略應用實例**                                                                 |
| ------ | --------------------------- | ------------------------------------------------------------------------------- |
| **S**  | **Substitute (替代)**         | 嘗試將簡單移動平均線 (SMA) 替換為 **EMA** 或[[加權移動平均線 (Weighted Moving Average)]]，觀察反應速度是否提升。 |
| **C**  | **Combine (合併)**            | 將「均值回歸」與「趨勢過濾指標」（如 **ADX**）結合，避免在強趨勢中逆勢接刀。                                      |
| **A**  | **Adapt (調整)**              | 調整布林通道參數，針對 **TSM (高波動個股)** 與 **ZN (低波動美債)** 使用不同的標準差倍數。                        |
| **M**  | **Modify (修改)**             | 改變計算週期，將原本的 **15m** K線改為 **HFT (高頻)** 的 Tick 數據，或修改止損邏輯（如從固定點數改為 ATR 波動率止損）。    |
| **P**  | **Put to another use (轉用)** | 將原本用於股票（TSM）的均值回歸模型，轉用到加密貨幣或外匯市場，測試其魯棒性。                                        |
| **E**  | **Eliminate (消除)**          | 簡化程式碼與過度擬合 (Overfitting) 的參數。例如：**KISS 原則**，刪除那些對勝率提升不明顯的複雜濾網。                  |
| **R**  | **Reverse (反轉)**            | 思考「反向策略」。如果均值回歸在特定區間失效，是否代表該區間適合轉為**趨勢追蹤 (Trend Following)**？                   |

## 技術實作與架構思考 (Cloud/DevOps Perspective)

當使用 **NestJS** 或 **Rust** 構建交易引擎時，SCRAMPER 可以導向具體的架構改進：

- **Substitute (替代):** 在雲端基礎設施中，將 Firebase 實時資料庫替換為 **Redis**，以換取極低延遲的訂單處理。

- **Combine (合併):** 利用 **Kubernetes (K8s)** 將策略回測模組與自動化 CI/CD 流水線結合，實現「回測成功即部署」。

- **Modify (修改):** 將同步的 API 請求修改為基於 **gRPC** 或 **WebSocket** 的非同步流處理，提升高併發下的系統性能。
