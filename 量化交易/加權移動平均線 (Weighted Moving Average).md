**加權移動平均線 (Weighted Moving Average, WMA)** 是移動平均線的一種進階變體。與簡單移動平均線 (SMA) 平等對待所有歷史數據不同，==WMA 對於**越近期的價格給予越高的權重**，而越遠期的價格權重則線性遞減==。

這種設計是==為了縮短指標的「滯後性 (Lag)」==，使其能更即時地反應當前的市場變化。

---

## 1. 數學計算原理

WMA 的權重是根據天數線性排列的。假設我們計算一個 5 日 WMA：

- 第 5 天（最新）：權重為 5
    
- 第 4 天：權重為 4
    
- ...
    
- 第 1 天（最遠）：權重為 1
    

**公式表示：**

$$WMA = \frac{P_n \times n + P_{n-1} \times (n-1) + \dots + P_1 \times 1}{n + (n-1) + \dots + 1}$$

其中 $P$ 為收盤價，$n$ 為週期天數。分母則是權重的總和（即 $\frac{n(n+1)}{2}$）。

---

## 2. 三種均線的特性對比

在您的交易系統開發中，選擇哪種均線取決於您對「信號速度」與「噪音過濾」的取捨：

|**類型**|**反應速度**|**平滑度 (過濾噪音)**|**主要用途**|
|---|---|---|---|
|**SMA**|最慢|最高|判斷長期趨勢支撐位。|
|**EMA**|快|中|捕捉趨勢轉折（指數型衰減權重）。|
|**WMA**|**最快**|**最低**|**極短線交易**，對價格變動極度敏感。|

## 3. 技術實作 (Rust / TypeScript)

由於 WMA 需要對數組進行加權求和，在處理大量時序數據 (Time-series) 時，應注意計算效率。

### Rust 邏輯範例

在開發您的策略引擎時，可以利用遞迴或迭代器來實作：

Rust

```
fn calculate_wma(prices: &[f64]) -> f64 {
    let n = prices.len();
    let weight_sum = (n * (n + 1)) as f64 / 2.0;
    
    let weighted_price_sum: f64 = prices.iter()
        .enumerate()
        .map(|(i, price)| price * (i + 1) as f64)
        .sum();

    weighted_price_sum / weight_sum
}
```

## 4. 在 TSM / ZN 策略中的應用建議

- **TSM (高波動個股)：** 使用 WMA 可能會產生較多「假訊號」，因為它對盤中的隨機波動過於敏感。建議搭配較長週期或結合 **RSI** 進行二次確認。
    
- **ZN (美債期貨)：** 債券市場趨勢通常較具連續性，WMA 能幫助您在趨勢啟動初期（例如利率決議公佈後）比 SMA 更早切入。
    

---

## 5. 邏輯校正：WMA vs EMA

- **線性 vs 指數：** WMA 的權重是「線性」下降的，這意味著超過 $n$ 天之前的數據權重會直接歸零；而 EMA 是「指數」衰減，理論上舊數據的影響力會一直存在但趨近於零。
    
- **應用取向：** 如果您的策略需要「徹底切割」舊數據的影響，WMA 是更專業的選擇。
